About

Designing durable interfaces for real people.

My work combines front-end engineering, accessibility reviews, and content structure so digital products are easier to understand and use.

Principles

How I approach the web

Start with structure

Clear headings, landmarks, lists, and form labels make pages easier to scan visually and easier to navigate with assistive technology.

Respect every input method

Keyboard, touch, mouse, voice, and screen reader users should all be able to move through the same content without hidden traps.

Keep pages resilient

Fast-loading markup, native controls, and progressive enhancement help sites remain useful across devices and connection speeds.

Experience

Focused accessibility practice

  1. Front-end accessibility consultant

  2. Design system contributor

    Built reusable components with keyboard behavior, focus management, and inclusive content guidance.

  3. Product interface developer

    Delivered responsive web pages with maintainable CSS, accessible forms, and SEO-friendly markup.

Working style

I like practical accessibility: specific issues, clear fixes, and implementation details that teams can actually maintain.
